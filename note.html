不保存,强制退出 12. diff difference diff 1.txt 2.txt 比较1.txt和2.txt文件 先删除d 下面aa目录 创建目录dirTxst 进入目录没在该目录中创建3个目录 a b c 创建文件1.txt 并写入数据 复制 1,txt 赋值到a 目录取名为2.txt 编辑2.txt 修改其中数据 删除一行 新添加一行 修改某一行的数据 在b目录下创建目录 c/d 删除c目录 git 命令的使用 所有的git命令都是以git 开头的 1.配置用户信息 git config --global user.name "你的用户名" git config --global user.email "你的邮箱" 2.初始化仓库 先进入目标文件夹 git init 自动生成一个 .git的隐藏文件夹 3.查看仓库的状态 git status 4.提交 git add 文件名 将该文件提交到暂存区 git add -A 将所有修改的提交到暂存区 git commit -m "提交说明" 提交到仓库去 5.git diff 查看两次的区别,当前与上一次提交时的区别 6.取消修改 1,没有提交到暂存区时,撤销工作区的修改 git checkout -- 文件名 取消还未add ,也没有commit的文件的修改 2,提交到暂存区(add) ,但是还没有commit 先回退到 add 将暂存区的修改取消 然后在取消工作区的修改 git reset HEAD 文件名 将add 取消 之后再使用git checkout -- 文件名 取消仓库的修改 已结 add,并取消commit了,取消仓库的修改 版本回退 git reset --hard 提交的id 7.查看存盘点(commit的历史记录) git log git log --pretty=oneline git reflog 查看所有提交的信息 包括回退的操作 8.gitstatus 会忽略空文件夹 提交也会忽略空文件夹 9.HEAD 村盘点,实现版本控制 git中把每一次的提交串成一条时间线,这个时间线就是一条分支,默认只有一条分支master ,也称之为主分支 ,HEAD 指向就是当前分支 10.查看当前分支 git branch 前面有* 表示是当前所在分支 11.创建分支 git branch 分支名称 12.切换分支 git checkout 分支名称 13.合并分支 git merge 分支名称 在master 分支下使用git merge v1.1 会将v1.1的修改合并到master中,v1.1中的内容不会改变 如果在v1.1中使用git merge master ,则将master的修改合并到v1.1中 1.在v1.1中修改1.txt的内容并提交 2.切换到merge分支,查看1.txt是否被修改 3.修改master分支中的1,txt 并提交 4.切换到v1.1分支,查看1.txt是否被修改 14,删除分支 git branch -d 分支名称 不能在当前分支删除自己 git branch -D 分支名称 强制删除
